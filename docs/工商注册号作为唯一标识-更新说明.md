# 🔄 重要更新：工商注册号作为唯一标识

## 📋 更新说明

根据你的正确建议，我们已将**工商注册号**作为公司的**唯一标识**，而不是公司名称。

---

## 💡 为什么这样设计？

### 工商注册号的优势
✅ **终身不变** - 公司成立时分配，永久有效  
✅ **全国唯一** - 18位统一社会信用代码，不会重复  
✅ **无法逃避** - 公司改名后注册号不变，历史记录永久关联  

### 公司名称的问题
❌ **可以变更** - 公司可以通过工商变更改名  
❌ **存在重名** - 全国有大量同名公司（如"XX科技有限公司"）  
❌ **容易混淆** - 分公司、子公司名称相似

## 🔧 具体修改内容

### 1. 后端API修改

#### 哈希生成逻辑
```go
// 修改前：基于公司名+注册号
func generateCompanyHash(companyName, registrationNo string) string {
    normalized := strings.TrimSpace(companyName) + "|" + strings.TrimSpace(registrationNo)
    return crypto.DeterministicHash(normalized)
}

// 修改后：仅基于注册号
func generateCompanyHash(registrationNo string) string {
    normalized := strings.TrimSpace(registrationNo)
    return crypto.DeterministicHash(normalized)
}
```

#### 必填字段调整

**举报公司（POST /api/v1/company/report）**
- ✅ `reporter_phone` - 举报人手机号（必填）
- ✅ `registration_no` - 工商注册号（**必填**，改为主键）
- ⚪ `company_name` - 公司名称（**选填**，仅用于显示）

**查询公司（GET /api/v1/company/query）**
- ✅ `registration_no` - 工商注册号（**必填**）
- ⚪ `company_name` - 公司名称（选填，用于辅助显示）

**公司申诉（POST /api/v1/company/appeal）**
- ✅ `contact_phone` - 联系人手机号（必填）
- ✅ `registration_no` - 工商注册号（**必填**）
- ✅ `reason` - 申诉理由（必填）

**陪审团投票（POST /api/v1/company/vote）**
- ✅ `registration_no` - 工商注册号（**必填**）
- ✅ `side` - 投票方（reporter/company）
- ⚪ `fingerprint` - 设备指纹（选填）

## 🎯 实际应用场景

### 场景1: 公司改名逃避举报
**问题**：某公司被举报后，通过工商变更改名  
**解决**：注册号不变，所有历史举报记录依然关联

**示例**：
- 2023年：公司名"黑心科技有限公司"，注册号 `91110108MA01234567`
- 2024年：改名为"良心科技有限公司"，注册号仍是 `91110108MA01234567`
- 查询结果：输入注册号，依然能查到2023年的举报记录

### 场景2: 同名公司混淆
**问题**：全国有100家"华为技术有限公司"（分公司）  
**解决**：每家公司的注册号不同，精准定位

**示例**：
- 华为技术有限公司（深圳总部）：`91440300279860880T`
- 华为技术有限公司北京分公司：`91110000633517654A`
- 华为技术有限公司上海分公司：`91310000607166726D`

### 场景3: 用户体验优化
**问题**：用户不知道注册号怎么办？  
**解决**：
1. 前端提示："可在企查查/天眼查查询"
2. 提供链接：`https://www.qcc.com/` 或 `https://www.tianyancha.com/`
3. 未来可对接API自动补全
