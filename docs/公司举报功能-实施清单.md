## 📊 数据库变更

### 新增表（3张）
1. `company_records` - 公司举报记录
2. `company_stats` - 公司申诉与投票统计
3. `company_watchlists` - 公司关注列表

### 新增索引（7个）
1. `idx_hash_company` - Hash索引（company_hash）
2. `idx_active_company_hash` - Partial索引（仅active记录）
3. `idx_company_stats_hash` - Hash索引（company_stats表）
4. `idx_company_records_tags` - GIN索引（JSONB标签）
5. `idx_company_watchlist_company` - Hash索引（关注列表-公司）
6. `idx_company_watchlist_user` - Hash索引（关注列表-用户）

### 数据迁移
无需手动迁移，GORM AutoMigrate会自动创建表和基础索引，自定义索引通过SQL DDL创建。

## 🐛 已知问题与限制

### 当前限制
1. 公司名称必须完全匹配（不支持模糊搜索）
2. 工商注册号为可选字段（建议强制要求以提高精度）
3. 证据文件仅支持图片和PDF
4. 单次最多上传9个文件
5. 描述最多5000字

### 待优化
1. 添加公司名称模糊匹配（需要全文搜索引擎）
2. OCR识别劳动合同关键信息

## 📞 技术支持

### 问题排查
1. **数据库连接失败**: 检查PostgreSQL是否启动，端口是否正确
2. **Redis连接失败**: 检查Redis是否启动，密码是否正确
3. **MinIO上传失败**: 检查MinIO是否启动，Bucket是否创建
4. **前端404错误**: 检查路由配置，确认组件已正确导入
5. **API 429错误**: 触发限流，等待1分钟后重试

### 日志查看
```bash
# 后端日志
tail -f backend/logs/app.log

# Docker日志
docker-compose logs -f backend
docker-compose logs -f postgres
docker-compose logs -f redis
docker-compose logs -f minio
```

### 数据库检查
```sql
-- 检查表是否创建
SELECT table_name FROM information_schema.tables 
WHERE table_schema = 'public' AND table_name LIKE 'company%';

-- 检查索引是否创建
SELECT indexname FROM pg_indexes 
WHERE tablename LIKE 'company%';

-- 查看举报记录数
SELECT COUNT(*) FROM company_records WHERE status = 'active';

-- 查看公司统计
SELECT COUNT(DISTINCT company_hash) FROM company_records WHERE status = 'active';
```
