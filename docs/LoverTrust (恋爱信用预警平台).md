# 产品需求文档 (PRD): LoverTrust (恋信/恋爱信用预警平台)

**版本：** v1.0 (储备方案)

**定位：** 恋爱关系中的信息不对称消除工具

**核心目标：** 提供基于脱敏数据的风险查询，通过“共识算法”识别潜在的情感欺诈者。

---

## 1. 产品逻辑架构

为了规避法律风险并提高可信度，平台采用 **“轻量化存储 + 零知识查询 + 多重证据权重”** 的逻辑。

- **数据录入端（免费）：** 鼓励受害者提供证据，通过结构化表单（时间、地点、行为分类）录入。
    
- **数据查询端（阶梯收费）：** * **初级查询：** 输入手机号，返回是否存在记录（免费/极低价）。
    
    - **深度查询：** 解锁脱敏后的证据链、行为分类、多方验证状态（付费 19.9元/次 或 会员制）。
        

---

## 2. 核心功能需求

## 2.1 风险信息录入 (Reporting)

- **主体标识：** * 姓名（脱敏存储，前端显示：张*某）。
    
    - 手机号（**强制全量输入用于Hash匹配**，前端显示：138****1234）。
        
    - 身份证（前6位+后2位）。
        
- **行为打标：** 勾选预设标签（如：隐瞒婚史、多线发展、经济纠纷、暴力倾向）。
    
- **证据附件：** 支持上传聊天记录截图、转账记录、录音（**统一存储于 MinIO** 对象存储中，仅在深度查询中展示缩略/模糊图）。
    
- **发布者保护（后台实名，前台匿名）：** 为防范恶意诬陷与虚假举报攻击（女巫攻击），强制发布者必须进行后台身份认证（如绑定手机或微信授权以记录哈希身份），但在前台展示和被第三方查询时，强制完全匿名化，平台不直接暴露发布者身份。
    

## 2.2 风险索引查询 (Inquiry)

- **精确匹配：** 用户必须持有完整的手机号或身份证号才能发起查询，禁止模糊搜索姓名（防止误伤同名者）。
    
- **匹配反馈：**
    
    - **0匹配：** “目前该号码尚无风险记录，请保持警惕。”
        
    - **1+匹配：** “发现 X 条关联记录，最近一次发生于 [城市]，风险等级：中/高。”
        

## 2.3 信用评分系统 (R-Score)

- **多维度权重：**
    
    - **证据权重：** 仅有文字描述（1分），有截图（5分），有判决书/报警回执（10分）。
        
    - **共识权重：** 不同用户对同一对象的独立举报，权重呈指数级增长。
        
    - **申诉扣减：** 若被举报人通过官方渠道申诉并提供反证，系统需降低显示权重。

## 3. 技术方案与数据隐私 (核心储备)

## 3.1 零知识隐私设计

作为架构师，建议采用以下存储策略以应对潜在的数据库审计：

- **PII 处理：** 手机号和身份证号绝不存明文。为防范彩虹表和暴力破解（中国内地手机号组合空间较小），必须放弃单次 SHA256。
    
    - `Search_Hash = Bcrypt(Mobile + System_Salt)`：作为慢速哈希算法，大幅增加暴力破解成本，真正实现“零知识查询”。
        
    - `Display_Mask = Mask(Mobile)`：用于前端展示。
        
- **静态层与缓存层：** H5 配合 SQLite 进行持久化资源管理。**必须引入 Redis 组件**，利用 Redis 存储高频查询缓存并精准地基于 `IP + 设备指纹 + 账户Token` 实现滑动窗口分布式限流，保护系统免受恶意拖库拖垮。
    

## 3.2 H5 交互流程

1. **首页：** 极简搜索框 + “恋爱避坑指南”信息流。
    
2. **搜索结果页：** 进度条加载（增强仪式感），显示风险雷达图。
    
3. **支付页：** 19.9 元解锁“详细背景调查报告”。
    
4. **详情页：** 脱敏后的时间轴、行为标签、模糊化的证据墙。

## 5. 法律避风港与风控机制

- **避风港声明：** 明确平台为 UGC（用户生成内容）技术服务商，内容真实性由发布者负责。
    
- **自动脱敏过滤：** 系统自动检测上传图片中的二维码、完整身份证号并进行模糊处理。
    
- **冷却机制：** 同一 IP 每天限制查询次数，防止同行利用爬虫大规模“撞库”窃取数据。
    
- **反向告警：** 如果同一对象在短时间内被高频查询，系统自动给相关发布者发送“该事件正受到关注”的通知。

## 6. 补充建议（项目储备思考）

1. **全球化适配：** 既然你的项目需要全球视野，建议在存储时增加 `Country_Code` 字段，不仅针对中国市场，海外华人圈对“杀猪盘”和“情感骗子”的防范需求同样巨大。
    
2. **SEO 策略：** 这种工具的流量来源主要是搜索。你可以针对“某某城市+渣男/渣女”做大量的长尾关键词页面，引导流量进入 H5。
    
3. **技术演进：** 未来可以引入 **DID (分布式身份)**，让被举报的行为记录在链上，真正实现“互联网永不磨灭的记忆”。
