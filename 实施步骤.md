第二阶段：后端核心 API 研发（Phase 2: 后端核心）
这一步涉及核心的业务逻辑、隐私计算与数据存储。
密码学与工具类实现
编写核心工具包：实现 Bcrypt 加密器（负责拼接 系统盐值(Salt) + 手机号 并进行哈希）、敏感信息脱敏器（例如将手机号变为 138****1234）。
数据库初始化（SQLite）
引入 Go 的 Sqlite 驱动（如纯 Go 实现的 modernc.org/sqlite 或者 gorm），根据需求文档编写并自动迁移表结构（主要是 risk_records 表）。
编写核心业务 API：
鉴权接口 (/api/auth)： 模拟手机号/微信实名认证换取 Token。
上报接口 (/api/report)： 解析表单，上传图片至 MinIO，并获取存储返回 URL。通过 Bcrypt 处理身份证/手机号入库。
查询接口 (/api/query)： 接收明文检索，实时计算出对应的 Bcrypt Hash，然后去 SQLite 里面校验比对，最后基于共识权重返回带星号的脱敏数据和评分。
接入 Redis 限流（反女巫攻击）
在 Gin 框架中实现一个全局的中间件，基于 IP + 设备指纹 的频率限制，如 1分钟内最多查 5 次。
